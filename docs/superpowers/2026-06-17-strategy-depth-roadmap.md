# 策略深度优化路线图 / Backlog

**日期：** 2026-06-17
**用途：** 汇总"提升**策略性**（玩家的有意义决策）"的工作项，分三档，含**代码事实锚点**、体量、为什么是策略、做法草图、协同关系。**每项动手前单独 brainstorm→spec→plan**（本文件只是详细 backlog，不是实现计划）。

体量：S=小（参数/UI 小改/接线）· M=中（跨文件、动流程）· L=大（子系统，需再拆）
状态：💡想法待设计

> **背景诊断（2026-06-15 代码审计）**：当前最强策略轴是**构筑**（装哪 5 个触发器 + 1 个门，类 Balatro 的"留什么"），choice 质量高。拖累整体策略性的是两类问题：① 已建好的深度被闲置/隐藏（reroll 写完没接线、boss 词条已 roll 不展示）；② 跑局**纯线性**、球数**完全固定**（无资源管理、无路径选择）。计分是**扁平叠加**、无条件链、无随状态成长。
> 计分公式：`score = base × (1 + mult_add) × Π(mul_mult)`（`scoring/scoring_engine.gd:5-28`），来源 ADD_BASE / ADD_MULT / MUL_MULT。
> 跑局：8 区 × 3 轮 = 24 轮线性（`run/run_manager.gd:62-86 advance()`），无分支。
> 解锁（`run/unlock_manager.gd`）：跨局元进度，决定**池子**，非局内选择。

## 第一档 · 便宜且立竿见影（激活已有深度 / 修半成品） ✅ 已实现推送 2026-06-18（commits 1a71b46/9d9938d/bebaa80）

> 共同点：成本低（S）、不动核心循环、把"已经存在但残废/隐藏"的策略变成真决策。**建议作为"棋盘持久"之后的下一步打包做。**

### 1.1 接通商店 Reroll（S）
- **现状**：`run/shop.gd` 的 reroll **逻辑已写完**——`reroll_cost()`（base 1，每次 +1：1→2→3…，`shop.gd:25-26`），`reroll_count` 进种子 tag 改变商品池；有单测（`tests/test_shop.gd:58-68`）。**但 HUD 没接按钮**（`view/hud.gd:113-140 show_shop()` 只有 4 个槽位按钮 + Continue，无 Reroll）。
- **为什么是策略**：「花钱重塑选项」是 roguelike 经济的核心决策（这把要不要赌一手更好的触发器？钱留着 reroll 还是直接买？）。现在完全白扔。
- **做法草图**：HUD 加 Reroll 按钮（显示当前 `reroll_cost`），接 `board_view` 调 `shop.reroll()`（沿用现有确定性 roll），扣钱、刷新 4 槽 UI。纯接线 + 一个按钮。
- **依赖/协同**：与 1.2（卖出回钱）一起，经济决策才完整。

### 1.2 触发器满槽可卖/换 + 卖出回钱（S~M）
- **现状**：触发器槽**硬上限 5**（`view/board_view.gd:706`），门只 1 个（新买替换旧的）。**满 5 个后再买的触发器被静默拒绝**（`board_view.gd:703-709` 未处理溢出，**疑似 bug**）。无卖出/回钱机制。
- **为什么是策略**：构筑是你最强的策略轴，但"买到满就锁死"让它变成一次性堆叠而非**持续取舍**。能卖/换 → "该丢哪个换哪个"成为每次购物的真决策；卖出回钱 → 经济多一个杠杆。
- **做法草图**：满槽时购买走"选一个替换"流程（UI 列已装备的 5 个供替换）；加"卖出"按钮回收部分金钱（如半价）。改 `buy_shop_slot` 装备逻辑 + HUD 装备栏可点。
- **风险**：先确认满槽静默拒绝是否真 bug（也可能有意），修复要同步 `equipped_triggers` 状态与测试。

### 1.3 商店预告下一个 BOSS 词条（S~M）
- **现状**：boss 词条（50% `ban_mult` / 50% `sparse` 移除 30% 钉）在**轮首就已确定性 roll**（`run/run_manager.gd:119-124 _roll_boss_mod`，`round_in_ante==2` 时 `_start_round` 里 roll），**玩家此时已可知**；应用在 `view/board_view.gd:364-383 _apply_boss_mod`。**但 UI 不展示**，"提前备战"这条策略是隐形的。
- **为什么是策略**：把"隐藏机制"变成**预判循环**——在 boss 轮前的商店里看到「下一个 BOSS：禁用 MULT」，玩家就会针对性调整构筑（禁 MULT 就别堆 MULT 钉/倍率，转投 base / ×mult）。低成本、高策略回报。
- **做法草图**：boss 轮的前一个商店（或 ANTE 内）把已 roll 的 `boss_mod` 在 HUD 显示成预告条；配合 1.1/1.2 让玩家能据此调整。
- **协同**：roadmap 里已有「Boss 预告 + 配额进度条」(S~M) —— **本项即其策略侧的细化/落地**，可合并。

## 第二档 · 局内资源管理（与"棋盘持久"叠加增益）

> 共同点：让"一轮 5 球"从固定流程变成**可管理的资源**，每球产生取舍。体量 M。

### 2.1 球数变成可管理资源：多球 / 存球 / 赌球（M）
> **赌球切片 ✅ 已实现推送 2026-06-22**（双倍或清零，G 键押注；commits 9fbf11a/304bd2c/8058d8d/beaeb54，268 测试全绿；`specs|plans/2026-06-18-gamble-ball*.md`）。**剩余**：多球（额外球扩展，LIFE 钉已 +1 发）、存球/提前收手。
- **现状**：每轮**固定 5 球**（`run/run_manager.gd:43 LAUNCHES_PER_ROUND := 5`，`spend_launch`/`launches_exhausted`）。唯一"管理"是效率——剩球进收入（`_payout` 的 `launch_bonus = launches_left`，`run_manager.gd:112-117`）。**无多球/存球/赌球**。
- **为什么是策略**：发球数是局内最自然的资源。
  - **额外球**（经典弹球 extra ball）：某钉/触发器命中给 +1 球 → "要不要为多打一颗 extra-ball 钉绕路"。
  - **提前收手**：达标后主动结束、把剩球势头/倍率带走 → 见好就收 vs 榨干。
  - **赌最后一球**：最后一球进入"翻倍但失败清零"模式。
- **协同**：**多球 + 棋盘持久密盘**强协同（多一颗球在被清开的盘上规划路线）；与 roadmap「漏斗接球=免费球」「豪赌/取舍」同源，可一起设计。

### 2.2 连击 / heat 可存可赌（M）
- **现状**：`_wall_heat` 只驱动视觉（霓虹）；连击 `_combo` 落定即按本发命中数结算（`scoring/combo_score.gd`，封顶 ×5）。没有"跨球累积、玩家选择兑现"的资源化。
- **为什么是策略**：做成一个**跨球累积的倍率/热度表**，玩家选何时"兑现"成分数爆发；空击（某球 0 命中）或漏球清零 → **每球贪婪续攒 vs 落袋为安**的张力。
- **做法草图**：把 heat/combo 提升为可累积 meta 值 + "cash out"动作（按钮/自动落定）；纯函数化（曲线/清零规则）便于 TDD。**会与现有 combo 计分交互，需一起重设计**（避免双重结算）。
- **风险**：与第一期"棋盘持久"的连击量级变化叠加，配额/曲线要一起调（见 `balance-tunables.md` 风险表）。

## 第三档 · 结构性大改（策略上限最高、范围最大，需单独立项）

> 共同点：体量 L，触及计分内核或跑局结构。**先把一二档做完、看实机反馈再上。**

### 3.1 乘区"成长引擎"（L）
- **现状**：倍率源基本**扁平叠加**——装 5 个 ADD_MULT = 固定 +1.0；**无条件链**（每个触发器独立监听事件，无 trigger→state→trigger）、**无随状态成长**的源（`scoring/trigger_runtime.gd`，`data/trigger_def.gd` 的 effect/value/listen_mask/condition）。
- **为什么是策略**：Balatro 式深度的真正来源是**随条件成长 / 相互放大的乘区**，不是扁平加法。例如：
  - "每清掉一个目标钉，本 run ×mult +0.1"（跨轮成长）。
  - "连续 N 球不空击，×mult 翻倍"（保持 streak）。
  - 触发器互相喂条件（命中 X 计数 → 喂养另一个按计数放大的源）→ 构筑出"引擎"。
- **做法草图**：扩 `TriggerDef`/`TriggerRuntime` 支持"读 run/round 状态做成长"的 effect 类型 + 条件计数器；新增几个成长型触发器 .tres。**改计分内核相关，需谨慎 + 充分单测**。
- **协同**：补上 1.2 的构筑取舍后，成长引擎让"装哪 5 个"从"堆数值"升级为"搭引擎"。

### 3.2 跑局分支地图（L）
- **现状**：8 区 × 3 轮**纯线性**（`run/run_manager.gd advance()`），无路径/难度/奖励选择。
- **为什么是策略**：Slay-the-Spire / Balatro 式**选下一间房**（钉多练分 / 钱多备战 / 精英高危高回报 / 商店）→ 整个 **run 级**策略层（路线规划、风险偏好、资源节奏）。
- **做法草图**：在 ANTE 之间插入"选节点"相位；节点类型 + 奖励/难度差异；RunManager Phase 流转改造 + 地图 UI。**子系统级，需再拆成多个 spec**。
- **协同**：roadmap 已有「整局路线图 run map」(M) 偏"目标感/可视化"；**本项是其"可选择分支"的策略化升级**，范围更大。

## 补充项 ✚（2026-06-17 追加，与"棋盘持久 + 路线 payoff"强协同）

### A. 弹内"出手顺序"策略 / intra-ball sequencing（M，价值高）
- **现状**：分数**落定才一次性结算**（`scoring/scoring_engine.gd`），故一发之内"先撞谁后撞谁"不影响结果。
- **为什么是策略**：改成**边飞边按顺序生效**——先撞 MULT 钉把倍率"点亮"，之后撞的钉才吃这个倍率；先撞普通钉就亏。每一杆变成"怎么让球先过倍率钉再扫分数钉"的小谜题。
- **协同**：与棋盘持久 + 特殊钉路线 payoff 三连强协同；少见但很深的弹球策略。

### B. 钉子"牌库"构筑 / peg deck-building（M~L，价值高）
- **现状**：玩家能装备触发器/门，但**不能塑造刷出来的钉子池本身**（钉池按 ante 渐进、随机）。
- **为什么是策略**：让玩家 **curate 会刷的钉子库**（加 2 颗 BOMB 进库 / 移除 POISON / 升 MULT 出现率）——Balatro 卡组构筑那一层。当前完全缺失。
- **协同**：塑造池子 → 持久盘上规划路线 → 串连锁，与 A、与第一期持久盘三连协同。
