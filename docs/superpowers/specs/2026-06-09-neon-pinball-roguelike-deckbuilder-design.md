# 设计文档：霓虹弹珠 × Roguelike 构筑

- **工作标题**：《OVERCLOCK / 超频》或《NEON CHAIN / 霓虹连锁》（占位，可改）
- **日期**：2026-06-09
- **赛道定位**：独立精品（Premium Indie）
- **一句话**：每一发弹球引爆一连串霓虹连锁，看着分数指数炸开——一款「再来一把」的弹珠引擎构筑 Roguelike。

---

## 0. 设计锚点（已确认）

| 维度 | 决策 |
|---|---|
| 赛道 | 独立精品，买断制，无内购 |
| 局内目标 | **刷分过线**（Ballionaire / Balatro 型）+ Balatro 式 Boss 修正词条（非战斗） |
| 题材调性 | **抽象霓虹 / 极简**（美术成本最低，最适配弹球打击感） |
| 首发平台 | **Steam PC 优先**，架构预留竖屏，便于移动移植 |
| 核心范式 | 弹珠物理手感 × 引擎构筑，三个前辈（Peglin/Ballionaire/Balatro）都没占的交叉点 |

## 1. 核心循环（局内 · 一轮 round）

1. 竖向棋盘，顶部可调角度发射，钉子散布其间，底部落球消失。
2. 玩家**调角度发射弹球**，球受确定性物理弹跳，击中钉子触发效果 + 计分。
3. 每轮给**有限发射数**与一个**目标分数(quota)**。
4. 发射数用完前达标 → 过关；没达标 → 单局结束（roguelike）。

> **核心张力**：瞄准技巧（手感）× 引擎构筑（策略）共同决定一发球能滚多大雪球——既不是纯抽奖，也不是纯数值。

---

## 2. 构筑系统（差异化核心）—— 四类积木

- **触发器（"芯片"，≈ Balatro 小丑）**：被动响应事件。例：「每击中蓝钉 +2 基础分」「球每弹跳 ×1.1 倍率」「一发球击中 ≥8 钉后落袋本发 ×3」。**槽位有限（约 5 个）**，是引擎的大脑。
- **钉子改造**：改棋盘本身——把普通钉变成 倍率钉 / 连锁钉(引爆相邻) / 增殖钉 / 炸弹钉，或加钉。改造在跑局中**永久改写棋盘**。
- **弹球改造（球种）**：改发射的球——分裂球 / 重球(多弹跳) / 幽灵球(穿透) / 磁力球(吸附)。每轮组一小袋球。
- **发射/入口改造（门）**：改球进场时的状态——普通 / 加速 / 散射 等门，装在入口通道上。占「入口槽」，详见 §6.5。

---

## 3. 计分数学（深度来源）

**本发得分 = Σ基础分 × 总倍率**。触发器分别喂「加法(基础分)」和「乘法(倍率)」两条线——经典的**加法 vs 乘法**张力让 build 分流。

### 三层数值结构（借鉴 Balatro）

| 层级 | 作用 | 稀有度 | 增长性 |
|---|---|---|---|
| +基础分 | 加底数 | 常见、便宜 | 线性 |
| +倍率 | 加到倍率上 | 中等 | 中期强 |
| **×倍率（乘倍率）** | 把倍率再翻倍 | **稀有、贵、有条件** | **指数 · 后期命脉** |

> **"啊哈"瞬间样例**：分裂球 + 「每弹跳 +0.2 倍率」+ 连锁钉 → 一发球裂成 8 颗、疯狂弹跳连锁、倍率滚到几十倍，满屏霓虹炸裂。

---

## 4. 加×乘平衡原则

1. **加法是地板、乘法是引擎，分工是故意的**：加法前期保命、后期跟不上指数 quota；乘法是唯一能追上指数墙的东西。玩家被迫从「铺基础分」过渡到「叠倍率」——这个转型本身就是 gameplay 弧线。
2. **用"稀缺 + 条件"锁住爆炸，而非削数值**：×倍率触发器 → 低出现率 + 带条件 + 占稀缺槽位。指数爆炸需要刻意搭建，自然受控。
3. **拥抱可控的爆炸**：偶尔搭出 1000 倍爆杀是核心爽点（破表幻想）。不硬封顶；靠 **quota 曲线 + 难度阶梯(stake)** 制衡高手，不靠 nerf 玩家。
4. **quota = 中位 build 期望火力的 ~75%**：开发期建数值表估算中位 build 每区期望分，把 quota 设为其 ~75%，给强 build 留 10–100 倍破表空间。最终数字靠 playtest。
5. **防哑火**：普通钉永远提供保底基础分；给足刷新/移除/卖出，让玩家能砍废卡、转型。
6. **示例数值（占位，实测再调）**：普通钉 +5 底分；倍率钉 +0.5 倍；稀有 ×倍率触发器 ×1.5（条件触发）。第 1 区 quota≈50，每区 ×1.6 增长，第 8 区≈900。

---

## 5. 手感设计原则（运气 vs 掌控）

核心哲学：**构筑决定「地板」，手感决定「天花板」，运气只影响中间的波动。**

1. **确定性物理（最关键）**：同角度 + 同棋盘 = 每次轨迹完全一致，绝不掺随机抖动 → 弹道可学习、可预判。
2. **瞄准预测线**：发射前显示前几跳虚线；显示长度 = 掌控感旋钮（可作难度选项）。
3. **飞行中微操（推台 / Nudge）**：一次有限的桌面微调，补救坏球，拉高技术上限。
4. **棋盘做漏斗/通道引导**：高价值口袋、刁钻角度才进的通道，让「好球」明显优于「乱打」。
5. **每轮多发球摊薄单发方差**：大数定律抹平运气，好瞄准持续抬高每发期望。
6. **构筑兜底**：引擎够强时平庸球也有分——build 是地板，手感是天花板。
7. **可选瞄准辅助**：硬核可关预测线/微操拿成就，新手可开。

## 6. 棋盘与关卡设计

### 棋盘解剖
四个功能区：**发射区 / 主钉阵 / 高价值风险区 / 漏斗导流**。每块棋盘都要有「安全打法」与「贪一把的高风险打法」的张力。

### 钉子摆位原则（别纯随机）
用**结构**制造有意义的瞄准选择：簇(cluster) 做连锁起爆点；导轨/通道(lane) 刁钻高回报；密度梯度造可预期弹道；对称（新手）vs 非对称（高难）。

### ⭐关键决策：钉子常驻为主 + 少数一次性
- **以「常驻」为主**（球弹过不消失）→ 弹道可学习、引擎可靠触发。
- **少数「一次性高价值钉」打中后耗尽/冷却** → 制造「优先打哪个」的目标排序与紧迫感。
- 常驻 = 可规划的引擎；一次性 = 临场取舍。两者混用兼得策略深度与清台爽感。

### ⭐棋盘即引擎
钉子改造在跑局中**永久改写棋盘**——玩家亲手把普通钉变成特殊钉，引擎**肉眼可见地长在战场上**（Balatro 的引擎在侧栏，本作直接长在弹珠台上）。

### 多样性（不手搓每一关）
**模板库 + 程序化填充**：手工设计若干布局原型（蜂巢/漏斗/双塔/螺旋），运行时随机填充钉子类型。按区(ante)调难度；Boss 词条改棋盘（移动钉/禁某钉/遮挡视野，零美术成本造变化）；布局原型随解锁开放。

### 一局节奏
开局 1–2 发铺垫/激活引擎 → 中段连锁滚雪球 → 最后 1 发 all-in 冲刺过线。保证「最后一发还能逆转」的戏剧性。

## 6.5 入口通道与门系统（第四类构筑：发射/入口改造）

球**不直接出现在棋盘上**，而是先经过一段**入口装置**改变进场状态，再释放进主钉阵：

```
选入口位置 → 入口通道(channel) → 门(gate) → 释放进主钉阵 → 物理弹跳计分 → 落底
 (三边滑选)     (一小段走廊)      (改变球状态)
```

### 入口位置（手感技术杆，免费、每发可换）
- **顶/左/右三条边均可作入口**，并可沿边**连续滑动选任意点**（已确认：三边任意滑动）。
- 顶部入 = 重力下落；左/右入 = 横扫/贴墙斜射 → 同一棋盘多种弹道几何。
- 属「瞄准技巧」层，不消耗资源；放大了左右通道、贴墙钉链类棋盘结构的意义。

### 门（Roguelike 构筑件，占「入口槽」，商店/掉落获取）

| 门 | 对进场球的变换 | 协同 |
|---|---|---|
| 普通门 | 原样放行 | 稳定基线 |
| 加速门 | 提高入场初速 → 更多弹跳/穿透 | 配「每弹跳+倍率」「穿透」流 |
| **变角散射门** | 单球入场角度种子化随机 | 偏「变数」 |
| **分裂散射门** | 1 球裂成扇形多球射出 | 偏「覆盖」，配铺钉流 |
| 充能门(扩展) | 每用一次积攒全局倍率 | 长线运营 |
| 镜像门(扩展) | 对称复制球到对侧入口 | 双线对称连锁 |
| 传送门(扩展) | 成对入口 A 进 B 出 | 空间链式 |

> 已确认：散射门做**变角 + 分裂两种**都进池。

### 门槽 / 串联（已确认）
- **MVP 单门槽**；后期解锁**「多门串联」**（通道串多扇门链式叠加，如 普通→加速→散射）作为高阶 build 维度。

### 与其他系统的咬合
- **手感**：入口位置(技术) + 门(构筑) 叠成完整战术（"从右边、用加速门、瞄这个钉簇"）。
- **预测线**：把门效果一并算入再画弹道；散射门显示**扇形范围**而非单线。
- **确定性**：散射/随机门走**种子化随机**（master seed 派生），每日种子与回放仍可复现。
- **棋盘设计空间**：左右入口让横向通道、贴墙连弹结构变得有策略价值。

### MVP 范围（入口/门）
三边滑动入口 + 普通门 / 加速门 / 变角散射门 / 分裂散射门（~4 门）+ 单门槽。串联、充能/镜像/传送门留作后期解锁。

## 7. 跑局结构

- 一局 ≈ **8 区(ante)**，每区 3 轮：小关、小关、**Boss 关**。
- **Boss 关 = Balatro 式修正词条（非战斗）**：如「禁用倍率钉」「只给 1 发球」「分数过半才计分」「钉子缓慢移动」——逼临场调 build。低成本、高变化，避开敌人 AI 内容黑洞。
- 目标分数指数增长；撑过第 8 区通关，叠**难度阶梯(stake)** 做长留。
- 节点间穿插：**商店**（买/刷新/移除/升级）、随机事件（小抉择/赌博）、宝箱。

---

## 8. 经济（纯局内单循环）

- 过关按「剩余发射数 + 存款利息」发钱（Balatro 式利息，鼓励攒钱滚利）。
- 钱用于商店。**无双货币战斗循环**，聚焦、好平衡。

---

## 9. 反枯燥五层机制（确保不做成干巴老虎机）

「数字变大」只是反馈层；不枯燥靠五层叠加：

1. **决策密度（玩法本体）**：每次商店/三选一都是有取舍的硬选择（槽位有限→必须砍）。
2. **横向多样性**：大池子 + 强协同 → 每局是一道不同的构筑谜题（倍率流/连锁流/分裂流/铺钉流）。
3. **quota 张力（赌注）**：指数增长的目标分在追杀你，制造生死张力。
4. **juice（感官烟花）**：连锁升调音效瀑布 + 屏震 + 慢动作 + ka-ching 逐项结算。
5. **破表幻想（爽点）+ 长留层**：偶发 1000 倍爆杀；解锁 + 难度阶梯 + 每日种子排行榜。

## 10. 差异化（护城河）

电梯陈述：**「Ballionaire 的引擎深度 × Peglin 的瞄准手感 × 一个长在战场上的空间引擎 × 霓虹视听」——三个前辈都没站的那个点。**

- **护城河 1 · 手感回归**：vs Ballionaire 的「丢球看戏」，我们把引擎深度 + 主动瞄准/微操缝起来。
- **护城河 2 · 棋盘即引擎**：vs 所有同类把引擎放侧栏，我们让引擎长在战场上，是全新 build 载体。
- **护城河 3 · 空间/几何构筑**：钉子有位置 → 协同是空间性的（相邻引爆、漏斗钉簇），Balatro 给不了、Ballionaire 用不上的维度。
- **护城河 4 · 视听签名**：霓虹极简 + 连锁升调瀑布，第一眼区分。
- **防换皮**：主动避开 Peglin 战斗 与 Ballionaire 全自动掉落，只吃没被认领的交叉地带；护城河 1/2/3 恰好都是最难抄的部分（手感、空间设计、视听调校）。

## 11. 元进度（买断制，无内购）

- **解锁制**：新触发器/球种/钉子（扩池）、新起手核心（像 Balatro decks）、新棋盘原型、新难度阶梯(stake)。
- **图鉴 + combo 挑战**：收集全触发器、达成指定邪道组合。
- **每日固定种子 + 排行榜**：确定性物理保证公平可复现。

---

## 12. 美术与 Juice（霓虹极简）

- 纯矢量几何 + shader 发光 + 粒子，**无角色动画**，单人可负担。
- **连锁升调音效瀑布 + 屏震 + 慢动作 + ka-ching 逐项结算**——juice 是本作生命线，第 1 阶段就要做。
- UI **竖屏友好**，为移动移植零返工。

---

## 13. 技术（已锁定）

- 引擎/语言：**Godot 4 + C#（.NET）**，IDE：**Rider**，版本管理：Git。
- **确定性物理**：自写定步长简化弹球（圆 vs 静态碰撞），不依赖 Godot 内置刚体——平衡、回放、每日种子、预测线都依赖它。
- **数据驱动**：触发器/钉子/球种/门 全用 Godot Resource(.tres) 配置，扩内容与调平衡不改代码。
- 详见配套技术设计文档：`2026-06-09-neon-pinball-tech-design.md`。

## 14. MVP 范围（砍到能验证乐趣的最小集）

1–2 棋盘原型、~15 触发器、~6 球种、~6 钉子、**~4 门 + 三边滑动入口（单门槽）**、**3 区（非 8）**、~4 Boss 词条、商店、1 难度、1 起手核心。

> **目标**：验证核心爽感 + 1–2 条 build 成立 + 手感。打磨 > 堆量。
