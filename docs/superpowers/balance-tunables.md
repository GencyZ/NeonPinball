## 目标钉 / 每轮目标感 📝（spec 已写，值在实现时落地；机制开关待你最终确认）

**数值（首版）**
| 参数 | 首版值 | 说明 |
|---|---|---|
| 目标钉数量 | `clamp(3+(区-1)/2, 3, 6)` | 1区3 / 3区4 / 5区5 / 7区6 |
| 目标钉 HP | `clamp(2+(区-1)/3, 2, 3)` | 1~3区 HP2，4~8区 HP3；**HP=1 即一击清** |
| 清光奖励金钱 | +5 | 清光所有目标钉的额外金钱 |

**机制开关**
| 开关 | 决定 | 备注 |
|---|---|---|
| 目标钉持久 | ✅ 跨本轮 5 发持久（填充钉照常重生）| 不持久则"清光"无意义 |
| 清光时机 | ✅ **只给奖励 + 高潮，不提前过关** | 清光设 `targets_done=true` + ALL CLEAR 高潮 + 奖励；**回合仍发完 5 球**，结束时按 `targets_done 或 够配额` 判定（用户 2026-06-15 定）|
| 与配额关系 | ✅ 双路：清光 或 够配额 | targets_done 在发完时也算赢 |

## 局内棋盘整轮持久 + 命中清除 + 补钉 📝（Phase 1，spec 待定）

spec：`specs/2026-06-17-persistent-board-strategy-design.md`

| 参数 | 首版值 | 位置（实现时） | 说明 / 改法 |
|---|---|---|---|
| `TOPUP_COUNT`（**头号旋钮**）| 3 | `view/board_view.gd` | 每发落定补几颗新钉。太小→第 3 发就近空、没钉可打；太大→棋盘不变稀、持久+规划失去意义 |
| 起始密度 `count` | `25+ante*2 .. 38+ante*3`（现值不变）| `view/board_view.gd` `_generate_pegs` | 配合 TOPUP 调，使 5 发自然变稀但不见底 |
| 过渡时长 | `PEG_EXIT_DUR=0.4 / PEG_ENTER_DUR=0.4` | `view/board_view.gd` | 改后只动被撞钉，是否觉得拖沓 |

**实机要盯的风险**
| 风险 | 说明 | 处置 |
|---|---|---|
| 连击量级上移 | 持久密盘可撞钉更多→单发连击/分数整体上移，或击穿配额、×5 太易封顶 | 实机后再回调 `combo_score` 曲线 / `quota_of`（**Phase 1 先不动**，见下决策②）|
| 棋盘空太快 | TOPUP/起始数没配好→后期近空盘 feel-bad | 先靠 `TOPUP_COUNT` 调；必要时 Phase 2 加密度地板 |

**Phase 1 三个待定决策（实现前可定，未定则用"倾向"）**
| 决策 | 选项 | 倾向 | 一句话 |
|---|---|---|---|
| ① TOPUP_COUNT 默认 | 2 / **3** / 5 | 3 | 先用 3 跑，实机调；本就是头号旋钮 |
| ② Phase 1 就动 combo 曲线？| 动 / **不动** | 不动 | 现在纯靠猜，先按原样实现、实机看了再调，免得调过头 |
| ③ 过渡保留阻塞？| **保留** / 去掉(非阻塞) | 保留 | 保留=改动最小最稳；实机觉得拖沓再去掉、做成打完即可瞄下一发 |
